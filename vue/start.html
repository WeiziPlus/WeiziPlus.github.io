# 框架(vue)

# 简介

  • 项目使用vue框架，UI框架为element-ui,网络请求使用axios
  • 项目对常用的组件进行了进一步封装
  • src/assets/sass/element-variables.scss 可以全局覆盖element-ui样式，可以自定义主题
  • store为官方store组件，如需使用请按照官方api
  • utils为工具类
    • axios 对axios进行了进一步封装
    • global_function 为常用的方法,并且挂载到Vue实例上面，调用方式
          this.$globalFun.successMsg('成功消息弹窗');
      1
    • global_variable 为常量,并且挂载到Vue实例上面,调用方式 this.$global.GLOBAL.base_url
  • views 为页面
    • common为侧边栏和顶部，无需操作
    • login 为登录页面
      • 登陆成功之后会在浏览器session中存放buttonMap,roleIdList,userInfo参数
      • buttonMap : 登录用户拥有的页面按钮map对象,用来显示隐藏页面按钮
      • roleIdList : 登录用户拥有的角色列表
      • userInfo : 登录用户的部分用户信息
    • main 为项目页面，通过功能管理页面进行配置的页面，全都基于main目录下
  • 路由规则
    • 功能管理页面进行创建菜单/按钮，并指定上级，填写path参数
    • 如果新创建的为菜单，需要在views/main目录下按照当前菜单的上级path依次创建path文件夹，最后一级创建Index.vue文件
      • 如，系统管理/功能管理,需要创建system/sysFunction/Index.vue
    • 如果为按钮，那么不需要操作
    • 功能管理如果创建菜单之后，需要重新退出然后登陆，这样侧导航栏才会有新创建的菜单显示
