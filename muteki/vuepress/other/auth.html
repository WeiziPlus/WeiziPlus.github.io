    # 权限部分说明(pc后台部分)

    • 说明 目前的思路为用户可以看到后台的 某个页面或者按钮，那么用户就拥有 访问该页面或者按钮触发的接口请求 的权限，前端通过 动态生成菜单栏和按钮 来展示权限，后端通过PcAuthorizationInterceptor拦截器拦截用户的访问请求
    • 权限设置
      • 如果 接口 不在任何 功能拥有的api 下，那么该接口只会判断是否登录，不做权限验证处理
      • 后台管理->系统管理->功能管理-> 编辑拥有api按钮
        • 可以设置为进入 此界面 或者可以操作 此按钮 所需要的接口url数组
        • 例如，进入 用户管理页面 需要查看 用户列表的接口权限，那么就设置 用户管理界面 拥有 用户列表的接口; 新增用户 需要 新增用户的接口权限,那么就设置 新增用户按钮 拥有 新增用户的接口
      • 动态展示按钮
              let {
         sysUser_add, sysUser_delete, sysUser_role, sysUser_resetPwd, sysUser_updatePhone, sysUser_updateStatus
      } = this.$globalFun.getSessionStorage('buttonMap');

        用户登录后会在 浏览器session 中存放用户拥有的 权限按钮列表 ,如果用户拥有新增用户的权限，那么 sysUser_add 的值为true，赋值给 vue的data 之后就可以动态展示/隐藏按钮
    • 权限配置
      • 角色管理模块给角色设置对应的功能即可
    • 页面展示/隐藏
      • 用户登录会自动获取拥有的角色，然后匹配到相应的权限，系统会自动渲染用户拥有的菜单列表和动态展示按钮
